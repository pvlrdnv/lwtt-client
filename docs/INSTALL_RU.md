# Установка LW TrustTunnel Client на Windows

Эта инструкция специально начинается с установки основного клиента TrustTunnel. LW TrustTunnel Client — это оболочка управления, а не замена `trusttunnel_client.exe`.

## 1. Сначала установите клиент TrustTunnel

Перед скачиванием проверьте тип системы Windows:

```text
Параметры → Система → О системе → Тип системы
```

Скачивать TrustTunnel Client нужно со страницы релизов официального проекта:

```text
https://github.com/TrustTunnel/TrustTunnelClient/releases
```

На этой странице откройте самый верхний релиз, то есть последний опубликованный релиз. В блоке **Assets** скачайте установочный архив TrustTunnel Client для вашей архитектуры Windows.

Обычно варианты называются примерно так:

- `trusttunnel_client-...-windows-x86_64.zip` — для большинства современных компьютеров с 64-разрядной Windows;
- `trusttunnel_client-...-windows-i686.zip` — для 32-разрядной Windows;
- `trusttunnel_client-...-windows-aarch64.zip` — для устройств на ARM-процессоре.

Для обычного современного ноутбука или ПК с Windows 10/11 чаще всего нужен вариант `windows-x86_64`.

Не скачивайте исходный код из кнопок `Source code`, если ваша цель — установить клиент на Windows. Нужен именно готовый архив клиента из блока **Assets** последнего релиза.

## 2. Создайте папку программы

Рекомендуемый путь:

```text
C:\Trusttunnel
```

Важно: все основные файлы должны лежать прямо в выбранной папке, а не во вложенной подпапке.

## 3. Распакуйте TrustTunnel

Распакуйте архив TrustTunnel в выбранную папку. После этого там должны быть как минимум:

```text
trusttunnel_client.exe
wintun.dll
```

Также могут быть дополнительные файлы TrustTunnel, например:

```text
setup_wizard.exe
LICENSE.txt
WINTUN_LICENSE.txt
```

## 4. Добавьте файлы LW TrustTunnel Client

Скопируйте содержимое папки `app/` из этого репозитория или из архива `dist/LWTT_Client_Manager_v4_15_public.zip` в ту же папку, где лежат `trusttunnel_client.exe` и `wintun.dll`.

После копирования в папке должны быть, среди прочего:

```text
trusttunnel_client.exe
wintun.dll
lwtt_tray_start.bat
lwtt_tray.ps1
lwtt_manager.ps1
lwtt_common.ps1
lwtt_start.bat
lwtt_stop.bat
```

## 5. Первый запуск

Запустите обычным двойным кликом:

```text
lwtt_tray_start.bat
```

Windows может запросить права администратора. Подтвердите запрос. Это нужно для работы сетевого туннеля и Wintun-адаптера.

## 6. Добавьте свой сервер

Профили серверов в публичную сборку не входят.

Откройте меню значка возле часов и выберите:

```text
Добавить сервер
```

Получите в тг-боте настройки сервера и файл сертификата `.pem`, если он требуется.

Затем вставьте целиком сообщение из тг-бота с настройками сервера и выберите файл сертификата `.pem`, если он был прислан.

## 7. Автозапуск

Чтобы запускать клиент автоматически при входе в Windows, запустите:

```text
lwtt_autostart_install.bat
```

Чтобы удалить автозапуск:

```text
lwtt_autostart_remove.bat
```

## 8. Обновление

1. Закройте приложение через меню трея.
2. Скопируйте новые файлы LWTT поверх старых.
3. Не удаляйте папку `profiles`, если хотите сохранить свои серверы.
4. Запустите `lwtt_tray_start.bat`.

## 9. Что не нужно переименовывать

Не переименовывайте:

```text
trusttunnel_client.exe
wintun.dll
```

Имена файлов LWTT также лучше не менять, потому что скрипты ссылаются друг на друга по этим именам.
